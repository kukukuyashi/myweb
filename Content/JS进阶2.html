4 JS 进阶 异步编程

学习 JavaScript 中异步编程的概念和实现方式，包括回调函数、Promise、async/await 等。

4.1 异步编程概述

JavaScript 是单线程语言，这意味着它一次只能执行一个任务。但是，JavaScript 提供了异步编程机制，允许在等待某些操作（如网络请求、文件读取等）完成时继续执行其他代码。

同步与异步

  • 同步：代码按照顺序执行，前一个任务完成后才能执行下一个任务
  • 异步：代码不需要等待前一个任务完成就可以继续执行
// 同步代码
console.log('开始');
console.log('中间');
console.log('结束');

// 异步代码
console.log('开始');
setTimeout(() => {
    console.log('延迟执行');
}, 1000);
console.log('结束');

4.2 回调函数

回调函数是作为参数传递给另一个函数的函数，当某个事件或条件满足时被调用。

基本用法

function fetchData(callback) {
    setTimeout(() => {
        callback("数据已加载");
    }, 1000);
}

fetchData(function(message) {
    console.log(message); // 数据已加载
});

回调地狱

当多个异步操作需要按顺序执行时，回调函数会嵌套多层，形成"回调地狱"，代码难以阅读和维护。

// 回调地狱示例
getData(function(a) {
    getMoreData(a, function(b) {
        getMoreData(b, function(c) {
            getMoreData(c, function(d) {
                // 多层嵌套，难以维护
            });
        });
    });
});

4.3 Promise

Promise是ES6引入的一种处理异步操作的方式，它代表了一个异步操作的最终完成或失败。

基本语法

Promise 对象有三种状态：

  • Pending（进行中）：初始状态
  • Fulfilled（已成功）：操作成功完成
  • Rejected（已失败）：操作失败
const fetchData = () => {
    return new Promise((resolve, reject) => {
        setTimeout(() => {
            const success = true;
            if (success) {
                resolve("数据已加载");
            } else {
                reject("加载失败");
            }
        }, 1000);
    });
};

fetchData()
    .then(message => {
        console.log(message); // 数据已加载
    })
    .catch(error => {
        console.error(error); // 加载失败
    })
    .finally(() => {
        console.log("操作完成");
    });

Promise 链式调用

Promise 的 then 方法返回一个新的 Promise，因此可以进行链式调用，避免回调地狱。

fetchData()
    .then(data => {
        console.log(data);
        return fetchMoreData(data);
    })
    .then(data => {
        console.log(data);
        return fetchMoreData(data);
    })
    .then(data => {
        console.log(data);
    })
    .catch(error => {
        console.error(error);
    });

Promise 静态方法

Promise 提供了一些静态方法，用于处理多个 Promise 对象。

// Promise.all：所有 Promise 都成功时才成功
const promise1 = Promise.resolve(3);
const promise2 = 1337;
const promise3 = new Promise((resolve) => setTimeout(resolve, 100, 'foo'));

Promise.all([promise1, promise2, promise3]).then((values) => {
    console.log(values); // [3, 1337, "foo"]
});

// Promise.race：第一个完成的 Promise 的结果
const promise4 = new Promise((resolve) => setTimeout(resolve, 500, 'one'));
const promise5 = new Promise((resolve) => setTimeout(resolve, 100, 'two'));

Promise.race([promise4, promise5]).then((value) => {
    console.log(value); // "two"
});

4.4 async/await

async/await是ES2017引入的一种处理异步操作的语法糖，它让异步代码看起来像同步代码一样。

async 函数

async 函数总是返回一个 Promise 对象。

async function hello() {
    return "Hello";
}

hello().then(value => console.log(value)); // Hello

await 表达式

await 只能在 async 函数内部使用，它会暂停 async 函数的执行，等待 Promise 完成后再继续执行。

const fetchData = () => {
    return new Promise((resolve) => {
        setTimeout(() => {
            resolve("数据已加载");
        }, 1000);
    });
};

async function main() {
    try {
        const message = await fetchData();
        console.log(message); // 数据已加载
    } catch (error) {
        console.error(error);
    }
}

main();

async/await 的优势

  • 代码更简洁，更易读
  • 可以使用 try/catch 进行错误处理
  • 可以使用 for、while 等循环语句

并行处理

使用 Promise.all 可以并行执行多个异步操作。

async function fetchAllData() {
    try {
        const [data1, data2, data3] = await Promise.all([
            fetchData1(),
            fetchData2(),
            fetchData3()
        ]);
        console.log(data1, data2, data3);
    } catch (error) {
        console.error(error);
    }
}

4.5 异步编程最佳实践

错误处理

始终使用 try/catch 处理 async/await 中的错误。

async function fetchData() {
    try {
        const response = await fetch('/api/data');
        const data = await response.json();
        return data;
    } catch (error) {
        console.error('获取数据失败:', error);
        throw error;
    }
}

避免阻塞

不要在循环中使用 await，除非需要按顺序执行。

// 不推荐：顺序执行，耗时较长
async function processItems(items) {
    for (const item of items) {
        await processItem(item);
    }
}

// 推荐：并行执行，耗时较短
async function processItems(items) {
    await Promise.all(items.map(item => processItem(item)));
}

总结

  1. 回调函数是最基本的异步编程方式，但容易形成回调地狱
  2. Promise 提供了链式调用，避免了回调地狱
  3. async/await 让异步代码看起来像同步代码，更易读
  4. 始终使用 try/catch 处理 async/await 中的错误
  5. 使用 Promise.all 并行执行多个异步操作，提高性能
